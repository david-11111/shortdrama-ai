# Agent Run 客户端 UI 设计方案

## 设计原则

参考 Claude Code / Codex 的交互模式：用户说目标，Agent 自己执行，用户观察过程。

核心体验：**不是填表单，是下达指令然后看 Agent 工作。**

---

### 交互细节

1. **项目选择器**
   - 下拉选择，显示项目名 + 缩略 ID
   - 记住上次选择（localStorage）
   - 如果只有一个项目，自动选中不显示下拉

2. **目标输入框**
   - 大文本框，placeholder: "告诉 Agent 你想做什么..."
   - 支持自然语言，例如：
     - "继续生成视频"
     - "从头开始，先写剧本再出图"
     - "只生成第 5-10 个镜头的关键帧"
   - 下方有快捷建议标签（根据项目当前状态动态生成）：
     - `继续生成视频 (18个待处理)` 
     - `生成预览小样`
     - `进入剪辑台`

3. **开始按钮**
   - 主色调按钮，大且明显
   - 点击后：POST /api/agent-runs → 拿到 run_id → 自动跳转观察页

4. **高级设置（默认折叠）**
   - 预算上限（数字输入）
   - 模式：step / autopilot（不暴露 preview，preview 是内部用的）
   - 普通用户看不到这些，只有展开才看到

5. **最近执行列表**
   - 最近 5 条 run 记录
   - 点击可进入对应 run 的观察页
   - 状态用颜色圆点：绿=完成，黄=运行中，红=失败

### 视觉风格

- 深色背景（#0d1117 类似 GitHub Dark）
- 主色调：蓝紫渐变（不是橙色，橙色太 debug 感）
- 字体：系统无衬线，标题稍大
- 输入框：深灰底 + 细边框，focus 时边框亮起
- 整体感觉：安静、专业、有科技感但不花哨

### 左侧状态栏

固定宽度 200px，始终可见。

内容：
- **Run 状态**：圆点 + 文字（running / completed / failed / paused / waiting_approval）
- **目标**：一行文字，截断显示
- **预算**：进度条 + 数字（已用/上限）
- **进度**：已完成任务数 / 总任务数
- **耗时**：从 run 创建到现在
- **操作按钮**：
  - 暂停（running 时可用）
  - 取消（running/paused 时可用）
  - 打开专家后台（跳转 /director/produce/:projectId）

### 中间事件时间线（核心）

这是用户花最多时间看的区域。

**事件类型和视觉表现：**

| event_type | 图标 | 颜色 | 展示方式 |
|------------|------|------|---------|
| decision | 🧠 | 蓝色 | Agent 做了什么决策 |
| tool_call | ⚙️ | 灰色 | 正在调用什么工具 |
| tool_result | ✓ | 绿色 | 工具返回了什么 |
| progress | ◐ | 黄色 | 等待中/轮询中 |
| risk | ⚠ | 橙色 | 遇到风险（backpressure 等） |
| error | ✗ | 红色 | 出错了 |
| writeback | 💾 | 紫色 | 数据已写入 |
| artifact | 📦 | 青色 | 产出了文件/媒体 |

**每条事件的结构：**

```
[图标] [标题]                              [时间]
       [详情文字，1-2 行]
       [可选：展开按钮 → 显示完整 JSON]
```

**实时行为：**
- 新事件从底部追加（类似终端日志）
- 自动滚动到底部（除非用户手动滚动了上面的内容）
- 正在进行的事件有呼吸动画（脉冲效果）

**折叠规则：**
- 连续多个相同类型的 progress 事件合并显示（如 "Seedance 轮询中... 第 12 次"）
- tool_call + tool_result 配对显示，result 缩进在 call 下面
- error 事件始终展开，红色高亮

### 右侧产物面板

宽度 250px，可折叠。

内容：
- **关键帧缩略图网格**（已生成的图片，2 列）
- **视频列表**（已生成的视频，带播放按钮和时长）
- **点击缩略图**：弹出大图/播放器
- **空状态**：显示 "等待 Agent 生成产物..."

## 页面 B 的状态变化

### running 状态
- 事件流实时更新
- 左侧进度条动态增长
- 产物面板随完成逐步填充

### completed 状态
- 顶部显示绿色横幅："Run 完成 · 生成了 6 个视频 · 耗时 4m 32s · 消耗 480 积分"
- 事件流停止更新
- 底部显示 "下一步建议"：
  - "继续生成剩余 12 个视频"
  - "进入剪辑台"
  - "生成预览小样"

### failed 状态
- 顶部显示红色横幅："Run 失败 · 原因：Seedance key pool 饱和"
- 失败事件高亮
- 底部显示 "重试" 按钮

### paused 状态
- 顶部显示黄色横幅："已暂停 · 等待你的指令"
- 显示 "继续" 和 "取消" 按钮

## 移动端适配

不需要完整移动端，但要保证：
- 平板横屏可用（隐藏右侧面板，左侧变顶栏）
- 手机上只显示事件流 + 状态，不显示产物面板

---

## 技术约束

1. **不 import produce/ 下任何组件**
2. **数据源只有两个**：
   - `GET /api/agent-runs/{run_id}/snapshot` — 页面加载时拉一次全量
   - `WebSocket project:{project_id}:agent_events` — 之后实时增量，但客户端必须按当前 `run_id` 严格过滤；服务端支持后优先切到 run 级订阅
   - snapshot、事件流和证据层必须有条数上限，长 run 通过分页/按需加载查看完整证据
3. **组件全部自建**，放在 `frontend/src/pages/director/agent-run/components/`
4. **状态管理用 composable**：`useAgentRunSnapshot.ts`
5. **不用任何 UI 框架**（不引入 Element/Ant/Vuetify），用原生 CSS + Vue
6. **暗色主题**，色板：
   - 背景：#0d1117
   - 卡片/面板：#161b22
   - 边框：#30363d
   - 主文字：#e6edf3
   - 次文字：#8b949e
   - 主色调：#58a6ff（蓝）
   - 成功：#3fb950
   - 警告：#d29922
   - 错误：#f85149
   - 紫色（writeback）：#bc8cff

## 与旧后台的关系

- 观察页左侧有 "打开专家后台" 按钮 → 跳转 `/director/produce/:projectId`
- 旧后台不加任何指向新客户端的入口（避免耦合）
- 两个页面共享同一个后端，但通过不同接口访问

## 第一版 MVP 范围

只做：
1. 启动页：项目选择 + 目标输入 + 开始按钮 + 最近执行
2. 观察页：状态栏 + 事件时间线 + 完成/失败横幅
3. 不做：产物面板（第二版）、暂停/取消（第二版）、移动端适配（第三版）
